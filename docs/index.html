Clinical failure modes
for healthcare AI agents.

Healthcare AI agents fail differently than general AI. A hallucinated medication status, a misrouted prior auth, a fabricated CPT code — these are patient safety issues. healtheval gives you named failure modes and the infrastructure to catch them.

Quickstart

No API key needed for deterministic checks.

from healtheval import run_eval

result = run_eval(
    "SCRIBE-001",
    run_llm=False,
    context="Metformin was discontinued on 2024-11-14.",
    agent_output="Patient is currently on metformin 500mg twice daily.",
)

print(result.final_verdict)  # FAIL
print(result.failed)         # True

Design Principles

Six decisions that shape everything in healtheval.

01

Deterministic first

Rules before LLMs. LLM reserved for semantic failures only.

02

Named failure modes

"Treatment Status Hallucination" beats "hallucination score 0.87".

03

No PHI, ever

All examples synthetic. No real patient data anywhere in the library.

04

Framework-agnostic

Works with any LLM, any agent framework, any observability layer.

05

Severity is clinical

Ratings reflect patient harm potential, not occurrence frequency.

06

Framework, not validator

An engineering tool. Not a certified clinical safety system.
